# Professional development

This folder is personal, not product. Everything under `docs/`, `specs/`, and the app
itself is the Spread Studio product and its process artifacts, built to demonstrate a
capability to a reviewer. This folder is different: it is a working study plan for
closing the real gaps between where the repo's PM (the human, not the agents) is today
and the target role archetype the repo was built to practice for.

The distinction matters. A spec candidate in `docs/product/spec-queue.md` is something
the repo will build. An item here is something the person needs to actually learn,
often outside any codebase: a licensing exam, a market mechanic, a domain convention no
amount of agent-built demo code substitutes for.

## Structure

- `domain-knowledge.md`: active trading, derivatives, and broker-dealer concepts to
  build real fluency in, not just simulate in code.
- `ai-agentic-workflow-skills.md`: tools and practices named or implied by the target
  role that go beyond what a portfolio repo alone can prove (Cowork, live spec review
  cadence at a real company, working with a real engineering team).
- `pm-craft.md`: PM fundamentals the repo can gesture at but not fully prove without
  a real product, team, and market: discovery at scale, competitive intelligence,
  go-to-market, stakeholder communication under real constraints.
- `learning-log.md`: dated entries. What was studied, what changed as a result, what
  is still open. Append-only, like the process playbook's lessons section, but for
  personal growth instead of repo process.

## Origin

This folder was split out of `docs/product/role-gap-analysis.md` on 2026-07-08. That
document mixed two different kinds of gaps: things the repo could build (now tracked
in `docs/product/spec-queue.md`) and things a person has to actually learn or
experience (tracked here). The original document's "Learning Notes" sections were the
seed for everything in this folder; its "What To Add" sections became spec queue
candidates.
